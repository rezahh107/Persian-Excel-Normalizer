# 🇮🇷 Persian Excel Normalizer

[![Python Version](https://img.shields.io/badge/python-3.9%2B-blue)](https://python.org)
[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](https://opensource.org/licenses/MIT)
[![PyQt6](https://img.shields.io/badge/GUI-PyQt6-brightgreen)](https://www.riverbankcomputing.com/software/pyqt/)
[![openpyxl](https://img.shields.io/badge/Excel-openpyxl-orange)](https://openpyxl.readthedocs.io/)

یک ابزار حرفه‌ای برای **پاک‌سازی و نرمال‌سازی متن فارسی** در فایل‌های اکسل (`.xlsx` و `.xlsm`).

این برنامه با تبدیل حروف عربی به فارسی، مدیریت هوشمند نیم‌فاصله (ZWNJ)، حذف کاراکترهای کنترلی نامرئی و یکسان‌سازی فاصله‌ها، داده‌های شما را برای استفاده در پایگاه‌های داده، سیستم‌های تحلیل متن و چاپ آماده می‌کند.

برای راهنمای کامل نصب و استفاده، [راهنمای جامع](GUIDE.md) را مطالعه کنید.

---

## ✨ قابلیت‌های کلیدی

- **تبدیل حروف عربی به فارسی**: تبدیل خودکار `ي` و `ك` عربی به `ی` و `ک` فارسی و سایر موارد مشابه.
- **مدیریت نیم‌فاصله (ZWNJ)**: قابلیت حفظ یا جایگزینی نیم‌فاصله (`U+200C`) با فاصله‌ی معمولی (قابل تنظیم در رابط کاربری و خط فرمان).
- **پاک‌سازی پیشرفته**: حذف کاراکترهای کنترل نامرئی (مانند ZWJ، LRM، RLM) و تبدیل فاصله‌های مجازی (NBSP) به فاصله‌ی عادی.
- **امنیت فرمول‌ها**: سلول‌های فرمولی با metadata خود `openpyxl` و guard محافظه‌کارانه‌ی مقدارهای شروع‌شونده با `=` شناسایی شده و دست‌نخورده باقی می‌مانند.
- **پشتیبانی ایمن‌تر از `.xlsm`**: فایل‌های macro-enabled با `keep_vba=True` بارگذاری می‌شوند تا آرشیو VBA هنگام ذخیره حذف نشود.
- **حفاظت از فایل اصلی**: برنامه از overwrite مستقیم فایل ورودی جلوگیری می‌کند و suffix خروجی باید با suffix ورودی یکی باشد (`.xlsx` به `.xlsx` و `.xlsm` به `.xlsm`).
- **Cancel امن‌تر**: اگر کاربر عملیات را لغو کند، خروجی نیمه‌نرمال‌شده ذخیره نمی‌شود.
- **دو رابط کاربری قدرتمند**:
  - **رابط گرافیکی (GUI)**: ساخته‌شده با PyQt6، با قابلیت کشیدن و رها کردن (Drag & Drop)، پردازش دسته‌جمعی، تم‌های روشن/تاریک و نمایش لحظه‌ای لاگ.
  - **رابط خط فرمان (CLI)**: مناسب برای اتوماسیون و استفاده در سرورها.
- **ذخیره‌سازی اتمیک (Atomic Save)**: خروجی ابتدا در یک فایل موقت در همان پوشه‌ی مقصد ذخیره و سپس با `os.replace()` جایگزین می‌شود تا ریسک فایل خروجی نیمه‌نوشته کاهش یابد.
- **پردازش موازی (Threaded)**: رابط کاربری در حین پردازش فایل‌های بزرگ، قفل نمی‌کند و کاملاً واکنش‌گرا باقی می‌ماند.

## 🛡️ نکات ایمنی Excel

- برای فایل‌های `.xlsm`، برنامه از `openpyxl.load_workbook(..., keep_vba=True)` استفاده می‌کند. این کار آرشیو VBA را حفظ می‌کند، اما اجرای واقعی macroها باید با Excel روی یک فایل نمونه بررسی شود.
- خروجی نباید همان مسیر ورودی باشد. برنامه in-place overwrite را رد می‌کند.
- suffix خروجی باید با ورودی سازگار باشد. برای حفظ macroها، فایل `.xlsm` باید به `.xlsm` خروجی داده شود.
- اگر عملیات Cancel شود، فایل خروجی برای workbook نیمه‌پردازش‌شده ذخیره نمی‌شود.
- `openpyxl` کل workbook را در RAM بارگذاری می‌کند. برای فایل‌های خیلی بزرگ، مصرف حافظه می‌تواند زیاد شود.
- قبل از پردازش فایل‌های مالی، اداری یا macro-enabled مهم، یک backup جداگانه نگه دارید و خروجی را با Excel بازبینی کنید.
- اگر فایل مقصد در Excel باز باشد، جایگزینی اتمیک ممکن است روی Windows شکست بخورد؛ در این حالت فایل اصلی نباید با خروجی نیمه‌نوشته جایگزین شود.
