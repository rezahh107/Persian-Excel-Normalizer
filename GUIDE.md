# راهنمای جامع Persian Excel Normalizer

این راهنما مراحل نصب، راه‌اندازی و استفاده از ابزار نرمال‌سازی فارسی اکسل را به‌صورت گام‌به‌گام توضیح می‌دهد.

---

## پیش‌نیازها

- **سیستم‌عامل**: Windows / Linux / macOS  
- **پایتون**: نسخه‌ی ۳٫۹ یا بالاتر  
- **مدیر بسته‌ها**: `pip`؛ بهتر است به‌روز باشد  
- **فضای دیسک**: حدود ۵۰ مگابایت برای نصب وابستگی‌ها  

---

## نصب و راه‌اندازی

### ۱. دریافت کد منبع

```bash
git clone https://github.com/rezahh107/Persian-Excel-Normalizer.git
cd Persian-Excel-Normalizer
```

> اگر از فایل ZIP استفاده می‌کنید، آن را استخراج کرده و وارد پوشه شوید.

### ۲. ایجاد محیط مجازی (پیشنهادی)

برای جلوگیری از تداخل با سایر پروژه‌های پایتون، یک محیط مجازی بسازید:

```bash
python -m venv venv
```

- **فعال‌سازی در Linux / macOS**:

  ```bash
  source venv/bin/activate
  ```

- **فعال‌سازی در Windows (Command Prompt)**:

  ```bash
  venv\Scripts\activate.bat
  ```

- **فعال‌سازی در Windows (PowerShell)**:

  ```powershell
  venv\Scripts\Activate.ps1
  ```

پس از فعال‌سازی، نام `(venv)` در ابتدای خط فرمان ظاهر می‌شود.

### ۳. نصب وابستگی‌ها

با استفاده از `pip`، کتابخانه‌های مورد نیاز را نصب کنید:

```bash
pip install PyQt6 openpyxl
```

> **نکته**: اگر در محیط‌های بدون دسترسی به اینترنت هستید، می‌توانید از فایل `requirements.txt` استفاده کنید؛ البته باید این فایل از قبل در پروژه یا بسته‌ی آفلاین آماده شده باشد.

## نحوه‌ی استفاده

برنامه به دو روش قابل اجراست: **رابط گرافیکی (GUI)** و **خط فرمان (CLI)**.

### ▶️ حالت گرافیکی (GUI)

این روش برای کاربران عادی و کسانی که با خط فرمان آشنا نیستند مناسب است.

```bash
python normalize_excel_gui.py
```

پس از اجرا، پنجره‌ی اصلی باز می‌شود. مراحل کار:

1. **انتخاب فایل(ها)**  
   - فایل‌های اکسل (`.xlsx` یا `.xlsm`) را با **کشیدن و رها کردن** در ناحیه‌ی ورودی قرار دهید.  
   - یا روی دکمه‌ی **Browse…** کلیک کرده و یک یا چند فایل را انتخاب کنید.

2. **تنظیم خروجی** (اختیاری)  
   - به‌صورت پیش‌فرض، خروجی در کنار فایل اصلی با پسوند `_normalized` ذخیره می‌شود.  
   - اگر فقط **یک فایل** انتخاب کرده باشید، می‌توانید مسیر خروجی دلخواه را با دکمه‌ی **Browse…** در بخش Output مشخص کنید.

3. **تنظیم گزینه‌ی نیم‌فاصله (ZWNJ)**  
   - تیک گزینه‌ی **Preserve ZWNJ** را بر اساس نیاز خود تنظیم کنید:  
     - **فعال (تیک خورده)**: نیم‌فاصله (`U+200C`) در متن حفظ می‌شود.  
     - **غیرفعال (تیک نخورده)**: نیم‌فاصله با فاصله‌ی معمولی جایگزین می‌شود.

4. **شروع پردازش**  
   - روی دکمه‌ی **Normalize Excel** کلیک کنید.  
   - پیشرفت پردازش در نوار پیشرفت و وضعیت‌های متنی نمایش داده می‌شود.  
   - در صورت نیاز، با دکمه‌ی **Cancel** می‌توانید عملیات را متوقف کنید.

5. **مشاهده‌ی لاگ**  
   - تمام رویدادها و خطاها در پنل پایین پنجره (Log) به‌صورت رنگی نمایش داده می‌شوند.  
   - با دکمه‌های **Copy Log** و **Clear Log** می‌توانید محتوای لاگ را کپی یا پاک کنید.

> **تغییر تم**: از منوی `View` می‌توانید بین تم روشن (Light) و تاریک (Dark) جابه‌جا شوید.

### ⌨️ حالت خط فرمان (CLI)

این روش برای اتوماسیون، اسکریپت‌نویسی و سرورها مناسب است.

```bash
python normalize_excel.py [INPUT] [OPTIONS]
```

#### پارامترها و گزینه‌ها

| پارامتر / گزینه | توضیح |
| --- | --- |
| `INPUT` | (اختیاری) مسیر فایل ورودی. اگر مشخص نشود، برنامه به‌طور خودکار اولین فایل `.xlsx` یا `.xlsm` را در پوشه‌ی جاری پیدا می‌کند؛ به‌جز فایل‌هایی که نامشان به `_normalized` ختم می‌شود. |
| `-o, --output` | (اختیاری) مسیر فایل خروجی. پیش‌فرض: `<نام_فایل_ورودی>_normalized.xlsx` |
| `--keep-zwnj` | (فلگ) اگر از این گزینه استفاده شود، نیم‌فاصله (`U+200C`) در متن **حفظ** می‌شود؛ در غیر این صورت با فاصله‌ی معمولی جایگزین می‌شود. |
| `-v, --verbose` | (فلگ) نمایش پیام‌های بیشتر در سطح DEBUG برای عیب‌یابی. |

#### مثال‌های کاربردی

**۱. پردازش خودکار فایل در پوشه‌ی جاری**

```bash
python normalize_excel.py
```

**۲. پردازش یک فایل مشخص با خروجی دلخواه**

```bash
python normalize_excel.py reports/data.xlsx -o results/clean_data.xlsx
```

**۳. پردازش با حفظ نیم‌فاصله و نمایش لاگ‌های دقیق**

```bash
python normalize_excel.py my_file.xlsx --keep-zwnj -v
```

**۴. استفاده در پایپ‌لاین‌ها**

```bash
find . -name "*.xlsx" ! -name "*_normalized.xlsx" -exec python normalize_excel.py {} \;
```

## معماری فنی (برای توسعه‌دهندگان)

- **هسته‌ی نرمال‌سازی** (`normalize_excel.py`):  
  - تابع `normalize_text` از `str.translate()` با **یک جدول ترجمه‌ی واحد** استفاده می‌کند که تمام جایگزینی‌های عربی→فارسی، حذف کاراکترهای کنترلی و تبدیل NBSP را در **یک پاس خطی (O(n))** انجام می‌دهد؛ این روش سریع‌تر از زنجیره‌ی چندین `replace()` است.  
  - دو جدول ترجمه برای حالت‌های **جایگزینی** یا **حفظ** ZWNJ به‌صورت پیش‌ساخته در زمان import ایجاد می‌شوند.  
  - سلول‌های فرمولی، یعنی سلول‌هایی که مقدارشان با `=` شروع می‌شود، شناسایی و از پردازش معاف می‌شوند.

- **مدیریت حافظه و ذخیره‌سازی**:  
  - `openpyxl` فایل را در RAM بارگذاری می‌کند. برای فایل‌های بسیار بزرگ، مصرف حافظه می‌تواند افزایش پیدا کند.  
  - برای جلوگیری از خرابی فایل هنگام ذخیره، از **ذخیره‌سازی اتمیک** استفاده می‌شود: ابتدا فایل در مسیر موقت مانند `<destination>.tmp~` ذخیره می‌شود، سپس با `shutil.move` جایگزین فایل نهایی می‌شود. در صورت بروز خطا، فایل موقت پاک می‌شود.

- **مدل هم‌روندی در GUI** (`normalize_excel_gui.py`):  
  - از الگوی استاندارد **Worker + QThread** پیروی می‌کند.  
  - شیء `NormalizerWorker` که از `QObject` ارث‌بری کرده، به یک `QThread` منتقل شده و پردازش در ترد پس‌زمینه اجرا می‌شود.  
  - تمام به‌روزرسانی‌های UI، شامل پیشرفت، وضعیت و لاگ، از طریق سیگنال‌های Qt به ترد اصلی ارسال می‌شوند؛ بنابراین رابط کاربری قفل نمی‌شود.  
  - قابلیت **لغو عملیات** با استفاده از یک flag که حین پردازش بررسی می‌شود، پیاده‌سازی شده است.

- **لاگ‌گیری**:  
  - دو لاگر مجزا برای `core` و `gui` تعریف شده‌اند تا لاگ‌ها از هم تفکیک شوند.  
  - در GUI، یک `QLogHandler` سفارشی لاگ‌ها را از هر تردی دریافت و با رنگ‌بندی سطح‌ها، یعنی DEBUG، INFO، WARNING و ERROR، در پنل متنی نمایش می‌دهد.
